# Handoff: МирГПТ Dashboard v2

## Что это
Дашборд продуктовых метрик МирГПТ. Чистый HTML + JS, **без бэкенда**. Источник данных — CSV‑файлы, которые аналитик выгружает из DBeaver и загружает в дашборд через шапку.

## О файлах
Это **рабочий прототип**, а не «дизайн‑референс». Задача в Claude Code — расширить парсеры под NEW‑метрики и добавить SQL‑запросы для DBeaver. UI/дизайн менять не нужно. Никакого API, авторизации, автообновления.

## Fidelity
**High‑fidelity.** Цвета, типографика, spacing, иконография, поведение — финальные. Поведение CSV‑аплоадера — времянка, заменяется на API (см. `SPEC.md` §4 Этап 2).

## Файлы

| Файл | Роль |
|---|---|
| `Dashboard v2.html` | Разметка + все стили (CSS inline). Открывается в браузере, работает с CDN. |
| `dashboard-v2.js` | Render‑логика (Chart.js), парсинг CSV (Papa Parse), переключатель периода, Tweaks. |
| `dashboard-v2-demo.js` | `window.DEMO` — фикстура данных. **Контракт API** — формат `/api/dashboard` должен совпадать. |
| `share.js` | Экспорт snapshot в один offline‑HTML (инлайнит Chart.js, Papa, шрифты base64). |
| `SPEC.md` | **Главный документ** — пошаговый план реализации, SQL для всех NEW‑метрик, acceptance criteria. |

## Поведение и интеракции
- **Период:** переключатель 7д/30д/90д/Всё в шапке — локальный фильтр массива `daily`. SQL аналитик запускает руками с нужными границами.
- **Hover:** на плитке появляется тень (`box-shadow: 0 0 0 1px var(--ink-2)`). Дельты (▲/▼) подсвечиваются `good`/`bad`.
- **Графики:** Chart.js с кастомной темой — никаких градиентов, только сплошные линии. Активность дублируется баром (ответы) + линией (пользователи).
- **Таблица промптов:** строки с пустым промптом (`null`/`без промпта`) подсвечиваются красной плашкой — это сигнал того, что юзер пишет напрямую без скилла.
- **«Поделиться»:** генерирует self‑contained HTML с зашитыми данными, шрифтами и Chart.js. ~550 КБ. Открывается без интернета.
